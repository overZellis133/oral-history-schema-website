# Contributing to Oral History Schema

Thank you for your interest in contributing to the Oral History Metadata Schema! This document provides guidelines and instructions for collaborating on the schemas.

## Ways to Contribute

### 1. **Share Feedback**
Have a question, suggestion, or found an issue? Open a GitHub Issue:
- Use the [Schema Feedback template](https://github.com/overZellis133/oral-history-schema-website/issues/new?template=schema-feedback.md)
- Be specific about which schema and field you're referring to
- Include your use case and any examples

### 2. **Propose Changes**
Want to modify or extend a schema? Create a Pull Request:
1. Fork the repository
2. Make your changes to the schema file(s) in `lib/schemas/`
3. Update any relevant documentation
4. Create a pull request with a clear description of:
   - What you changed and why
   - How it affects existing implementations
   - Examples of the new structure (if applicable)

### 3. **Join Discussions**
Participate in community discussions:
- [GitHub Discussions](https://github.com/overZellis133/oral-history-schema-website/discussions) - General questions and design discussions
- Comment on existing issues and pull requests

### 4. **Improve Documentation**
Help make the schemas more understandable:
- Fix typos or clarify descriptions
- Add examples
- Improve guides and tutorials
- Translate documentation

## Schema Files

The main schema files are located in:
- `lib/schemas/oral_history_schema.json` - Interview schema
- `lib/schemas/collection_schema.json` - Collection schema

## Guidelines

### Schema Changes

When proposing schema changes:

1. **Backward Compatibility**: Consider how changes affect existing implementations
2. **Documentation**: Update relevant documentation files
3. **Examples**: Provide example JSON that demonstrates the change
4. **Use Cases**: Explain the real-world use case that requires the change

### Code of Conduct

- Be respectful and constructive in all interactions
- Focus on the technical merits of proposals
- Help others understand your perspective
- Welcome feedback on your contributions

## Getting Started

1. **Explore the Schemas**: Visit the [schemas page](/schemas) to understand the current structure
2. **Review Examples**: Check out the [examples](/examples) to see real-world usage
3. **Read Documentation**: Review the [documentation](/docs) for implementation details
4. **Start Small**: Begin with feedback or small documentation improvements

## Questions?

- Open a [GitHub Issue](https://github.com/overZellis133/oral-history-schema-website/issues) for questions
- Join [GitHub Discussions](https://github.com/overZellis133/oral-history-schema-website/discussions) for community conversations
